 第一次课设计逻辑

  核心矛盾：学生零基础 +
  课程终点是做出上线产品 →
  第一课必须解决"我能做到吗"的信心问题， 
  而不是上来就堆知识。

  三件事并行：

  ┌───────┬──────────────────────────┐   
  │ 目标  │          怎么做          │   
  ├───────┼──────────────────────────┤   
  │ 消除  │ 先展示8周终点（成品），  │   
  │ 恐惧  │ 再讲路径；唯一的动手只需 │   
  │       │ 会打字                   │   
  ├───────┼──────────────────────────┤   
  │ 认知  │ 用真实数据+案例对比，让  │   
  │ 重建  │ 学生自己说出"宣传模式"的 │   
  │       │ 问题所在                 │   
  ├───────┼──────────────────────────┤   
  │ 第一  │ 10分钟AI受众模拟体验，当 │   
  │ 个成  │ 堂完成，人人都能做       │   
  │ 就感  │                          │   
  └───────┴──────────────────────────┘   

  90分钟结构：
  1. 5分钟 — 破冰举手提问（摸底 + 定调） 
  2. 20分钟 — 路线图 +
  考核说明（先看终点再出发）
  3. 25分钟 — 三个失效模式 + 真实数据冲击
  4. 25分钟 — 叙事模式 +
  霍夫斯泰德文化维度 + 最大公约数原则    
  5. 10分钟 — 第一次AI受众模拟动手       
  6. 5分钟 — 组队 + 课后三个小任务       

  刻意的设计决定：
  - 第一课不讲任何工具操作，防止技术焦虑 
  淹没认知框架
  - 霍夫斯泰德只讲最实用的2个维度，不贪全
  - 课后作业设计成"找视频+初步选题"，零门
  槛，但能推动下一课的讨论
