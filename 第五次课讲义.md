### 【第二节】视频大模型 vs 程序化框架（25分钟）

#### 2.1 两种工具，两种思维

**教师话术**：

> 上节课的海螺AI、即梦、可灵，本质上是**生成器**。你给它一段文字描述，它"凭空造出"一段视频。
> 今天要学的 Remotion，本质上是**组装线**。你给它素材和指令，它按照你写的代码精确地拼装成视频。
>
> 打个比方：
> 视频大模型像一个天才画家，你说"画一幅沙漠驼队"，他挥笔而就，每次画出来都不一样。
> Remotion 像一条印刷流水线，你设计好模板，放进去100份不同的数据，出来100份格式统一的成品。
>
> **画家和流水线不是竞争关系，是上下游关系。**

#### 2.2 核心差异对比

**板书/PPT**：

| 维度 | 视频大模型（海螺/即梦/可灵） | 程序化框架（Remotion） |
|------|--------------------------|---------------------|
| 底层逻辑 | AI概率预测（每次结果不同） | 代码逻辑（每次结果完全一致） |
| 核心能力 | 无中生有，生成视觉素材 | 精确组装，批量输出成品 |
| 文字处理 | 经常乱码、变形 | 完美支持所有字体和排版 |
| 多语言扩展 | 很难保持画面一致 | 换一个JSON文件，画面100%一致 |
| 精确控制 | 模糊控制（"大概这个感觉"） | 像素级、帧级精确控制 |
| 学习门槛 | 低（会写提示词就行） | 高（需要懂React代码） |
| 单次制作成本 | 高（GPU算力按次计费） | 低（CPU渲染，接近免费） |
| 修改成本 | 高（改一点就要重新生成） | 极低（改一行代码，全部同步更新） |

**教师话术**：

> 看这张表的最后两行。
> 如果你只做一条视频，大模型更快更省事。
> 但如果你要做30条、100条、1000条格式统一的视频，程序化框架的优势是碾压级的。
>
> 丝路文化国际传播恰好是后者。你们期末要做的不是一条爆款视频，而是一套系统性的内容产品。

#### 2.3 互补关系：AI生产"肉"，Remotion搭建"骨骼"

**板书/PPT**：

```
最佳实践：AI + Remotion 协作分工

AI大模型负责：                  Remotion负责：
├── 生成背景视频片段              ├── 组装所有素材到时间线
├── 生成配音（多语种）            ├── 叠加精确的标题和字幕
├── 生成背景音乐                  ├── 添加品牌Logo和片头片尾
└── 生成风格化图片素材            ├── 控制转场动画
                                  ├── 批量输出多语言版本
                                  └── 一键修改全部视频
```

> **一句话总结：AI解决"原材料从哪来"，Remotion解决"怎么批量组装成产品"。**

**课堂讨论**（3分钟）：

> 回想你们上节课用海螺生成的视频片段。如果要把它变成一条完整的、带标题字幕Logo的成品视频，你用剪映手工做需要多久？如果要做10个语言版本呢？

### 【第四节】Remotion 核心概念（25分钟）

#### 4.1 什么是 Remotion

**教师话术**：

> Remotion 是一个基于 React 的程序化视频框架。当前最新版本是 v4.x。
> 核心理念：**用写代码的方式制作视频**。
>
> 传统视频剪辑是"拖拽时间线"，Remotion 是"用数据和代码驱动时间线"。

```
React组件 → 浏览器逐帧渲染 → FFmpeg编码 → MP4视频
```

> 这意味着四件事：
> - **可复用**：同一模板，换数据就能生成新视频
> - **可批量**：一次编写，自动生成100个语言版本
> - **可版本控制**：视频逻辑存在代码里，用Git管理，改动有记录
> - **可自动化**：和AI生成的素材（图片、配音、字幕）无缝对接

#### 4.2 三个核心概念

**概念一：Composition（合成）——定义一个视频项目**

> Composition 就是一个视频的"身份证"，声明了尺寸、帧率、总时长。

```jsx
<Composition
  id="SilkRoadCities"
  component={SilkRoadCities}
  durationInFrames={300}  // 10秒 × 30帧
  fps={30}
  width={1920}
  height={1080}
/>
```

> 这段代码说：这个视频叫"SilkRoadCities"，1920×1080分辨率，30帧/秒，总共10秒。

**概念二：Sequence（序列）——控制谁在什么时候出场**

> Sequence 决定了视频里每个元素的出场时间和持续时长。

```jsx
<Sequence from={0} durationInFrames={90}>
  <TitleCard text="丝路城市巡礼" />
</Sequence>
<Sequence from={90} durationInFrames={120}>
  <CityCard name="长安" description="丝绸之路的东方起点" />
</Sequence>
<Sequence from={210} durationInFrames={90}>
  <CityCard name="敦煌" description="河西走廊的艺术宝库" />
</Sequence>
```

> 前3秒是标题，接着4秒展示长安，再3秒展示敦煌。就像写剧本一样，只不过用代码写。

**概念三：interpolate（插值）——让画面动起来**

> interpolate 是动画的核心。它把"当前是第几帧"翻译成"透明度/位置/大小应该是多少"。

```jsx
const frame = useCurrentFrame();

// 前30帧（1秒）内，透明度从0渐变到1
const opacity = interpolate(frame, [0, 30], [0, 1], {
  extrapolateRight: 'clamp',
});

// 地图路线动画：随帧数推进，路线逐渐延伸
const progress = interpolate(frame, [0, 150], [0, 1], {
  extrapolateRight: 'clamp',
});
```

> `interpolate(frame, [起始帧, 结束帧], [起始值, 结束值])` —— 就这一个函数，能做出渐显、移动、缩放、路线延伸等各种动画。

#### 4.3 类比总结

**板书/PPT**：

```
传统剪辑（剪映/Premiere）      Remotion
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
时间线 ← 手动拖拽              时间线 ← Sequence 代码
关键帧 ← 手动设置              关键帧 ← interpolate 函数
项目文件 ← .prproj              项目文件 ← React 代码 + JSON 数据
导出 ← 点"导出"按钮            导出 ← 命令行 npx remotion render
批量 ← 不支持                  批量 ← 循环 + 数据库
```

> 不需要你们现在就会写React。今天的目标是**理解这三个概念**，实操环节你们在模板基础上修改就行。

### 【第五节】Remotion + AI 工作流（15分钟）

#### 5.1 完整工作流

**板书/PPT**：

```
课题策划
  ↓
脚本撰写（人工 / AI 辅助）
  ↓
AI 素材生成（并行）
  ├── 画面：可灵/海螺生成历史场景片段
  ├── 配音：ElevenLabs/Fish Audio 生成多语种旁白
  ├── 配乐：Suno AI 生成丝路风格音乐
  └── 字幕：AI翻译 + 人工校准
  ↓
Remotion 组装
  ├── 导入AI素材（视频、图片、音频）
  ├── 编排时间线（Sequence）
  ├── 添加动画（interpolate）
  ├── 叠加标题、字幕、Logo
  └── 参数化多语言版本（切换JSON配置）
  ↓
渲染输出
  ├── 本地：npx remotion render
  └── 云端：Remotion Lambda 并行渲染
  ↓
适配分发
  ├── YouTube 横版 16:9
  ├── TikTok 竖版 9:16
  └── Instagram 方形 1:1
```

#### 5.2 AI视频工具全景

| **组装编排** | **Remotion**、Motion Canvas | **程序化视频合成与批量输出** | **今天** |

> 前三层解决"原材料从哪来"，第四层解决"怎么组装成产品"。

## 五、教学注意事项

**关于技术门槛**：本课是全课程技术要求最高的一次。学生不需要会写React，只需要能看懂JSON数据格式、能修改简单的数值和文字。教师准备好完整的模板项目是关键，学生在模板上改数据就行。

**关于环境搭建**：Node.js 安装是最大的不确定因素。课前一周在微信群通知学生安装，并附安装教程链接。教师准备好备用方案：如果个别学生安装失败，可以两人共用一台电脑。

**关于课堂节奏**：第二节（对比分析）和第四节（核心概念）是认知核心，务必讲透。实操环节如果时间紧张，可以压缩学生练习时间，保证教师演示完整走通一遍。

**关于网络环境**：`npx create-video@latest` 需要下载 npm 包，依赖网络。教师课前准备好已安装依赖的模板压缩包，学生解压后直接 `npx remotion studio` 即可，不依赖课堂网络。

**关于与上节课的衔接**：反复强调"上节课学了素材生成，这节课学组装"。让学生把上节课的海报导入Remotion模板是最好的衔接点。
